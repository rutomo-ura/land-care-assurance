Proposal Focus

The work separates three things that are currently easy to blend together: invalid assignments, infeasible assignments, and contractor non-compliance.

1Trusted monthly assignment denominator.
2Correct Active and Request Only completion metrics.
3Contractor compliance scorecard with fair context.
4Map view by month, contractor, status, and exception.
5Bundle optimization using geography, area, capacity, and history.
6Survey form pilot path for ArcGIS or open-source web.

Workflow Boundary

The current monthly cycle is partly scheduled and partly manual. The proposal adds assurance gates around this cycle before attempting a full platform change.

Step Status Proposal Implication
15th, 3:00am: bundle assignment CSV Scheduled through `bundle_assignment_creation.py` Add pre-15th universe QA, ownership checks, and exclusion logging.
15th, 3:15am: assignments to SQL Scheduled through `BundlesDriveToSQL.py` Make assignment records carry valid denominator and exception fields.
15th, 3:30am: Regrid download Scheduled, but dependent on Regrid export setup Wrap Regrid behind a survey ingestion contract while piloting alternatives.
18th, 3:45am: survey upsert Scheduled through `SurveysDriveToSQL.py` Add post-18th compliance review and open-assignment monitoring.
Contracts, NetSuite, Regrid export URL Manual or not fully automated Track as operational risks and prioritize the Regrid URL in Phase 3.

Three Phases

Click through the phase details. Each phase has a concrete output and a decision it supports.

Monitoring

  • Reconcile source assignment count to dashboard count.
  • Add data freshness for assignments, surveys, and ownership.
  • Track open, overdue, returned, and exception counts.

Map View

  • Filter by month, contractor, completion status, and ownership.
  • Show assigned vs returned parcels spatially.
  • Use the map to expose territory and workload patterns.

Data Integrity

  • Flag sold, transferred, duplicate, stale, or non-URA/PLB parcels.
  • Log exclusion reasons before monthly assignment.
  • Document the denominator and ownership rules.

Correct Metrics

  • Split Active and Request Only completion.
  • Report valid-assignment rate and exception rate beside completion.
  • Build contractor compliance metrics with fair denominators.

Visual Product

  • Publish scorecards in the web app and/or ArcGIS Experience Builder.
  • Show contractor workload by parcel count and acreage.
  • Compare current and proposed assignment scenarios.

Optimization

  • Bundle by geography, area, capacity, maintenance level, and history.
  • Use past completion patterns as a transparent score.
  • Pilot recommendations before automating assignments.

Survey Form

  • Design land care provider form fields and validation.
  • Capture parcel, period, contractor, status, photo, location, and issue reason.
  • Prevent duplicate and unmatched submissions.

ArcGIS Path

  • Evaluate Survey123, Field Maps, and Experience Builder.
  • Use hosted feature layers for operational adoption.
  • Align the survey form with the same metric data contract.

Regrid Decision

  • Keep Regrid only if it maps cleanly to the contract.
  • Wrap or reduce manual export steps where possible.
  • Recommend keep, wrap, reduce, or replace after pilot evidence.

Two-Month Roadmap

The plan is designed to produce useful outputs every two weeks and still fit inside a realistic two-month delivery window.

Weeks 1-2

  • Reconcile assignments and returns.
  • Define metric logic.
  • Create exclusion reasons.
  • Build first map data contract.

Weeks 3-4

  • Build monitoring view.
  • Add contractor filters.
  • Show Active vs Request Only.
  • Publish scorecard draft.

Weeks 5-6

  • Define bundle constraints.
  • Score historical completion.
  • Compare current vs optimized scenarios.
  • Recommend pilot rules.

Weeks 7-8

  • Design survey form workflow.
  • Test ArcGIS and web intake options.
  • Write Regrid transition recommendation.
  • Package final implementation backlog.

Metric Framework

The proposal replaces one blended percentage with a diagnostic operating view.

Metric Definition Why It Matters
LandCare success rate Valid returned surveys divided by valid expected assignments. Measures the core improvement goal without invalid parcels inflating the denominator.
Contractor compliance rate Valid returned surveys divided by valid assignments expected from each contractor. Creates fair contractor comparison after exclusions and workload context.
Valid-assignment rate Valid assignments divided by all candidates before exclusions. Shows whether the pipeline is assigning the right parcels.
Exception rate Assignments excluded or flagged due to ownership, stale tag, duplicate, or eligibility issue. Prevents metric improvement from hiding data-quality problems.
Optimization score Expected completion using geography, acreage, capacity, and historical patterns. Helps assign bundles that are more likely to be completed.

Product Path

The safest architecture is one data contract with two possible surfaces: open-source web for analytic speed and ArcGIS for field adoption.

Open-Source Web App

  • Fast custom analytics and scenario comparison.
  • Useful for proposal demos and optimization views.
  • Can extend the current Leaflet prototype.

ArcGIS Experience Builder

  • Strong hosted-layer and staff adoption path.
  • Pairs naturally with Survey123 and Field Maps.
  • Best for operational field workflow rollout.
